Visualization

Only plots batch plots over sets and runs.

Features:

- multiple plotting styles (flaviz_parquet.py, config.json)


Comparing our final serverless, DuckDB-powered batch visualizer (flaviz_parquet.py) against the official FLAViz Library, we can identify structural differences and features missing from our pipeline.

The original FLAViz library was designed to handle high-dimensional macroeconomic agent-based simulation data across multiple levels: Agent Types → Parameter Sets (Calibrations) → Monte Carlo Runs (Random Seeds) → Iterations (Time) → Individual Agent Instances → Multi-Dimensional Variables.

The core features missing from our script fit into three primary categories:

1. Missing Architectural & Data Transformation Features

    Monte Carlo Run Independence (ByGroup operations): Our DuckDB queries instantly aggregate data by grouping strictly on the time step (_ITERATION_NO). The original FLAViz script uses complex Pandas groupby configurations to slice or isolate specific individual random seeds (Monte Carlo runs) or parameter sets before mapping. Our script groups all random seeds in a subfolder tree together into a single averaged line/band.

    Arbitrary Index Slicing: The original FLAViz library allows the user to filter data frames using explicit logical coordinate slices across the hierarchy (e.g., “Only analyze Households who have a tracking ID between 100 and 200, across Monte Carlo runs 1 through 5”). Our script assumes an all-or-nothing aggregation per subfolder dataset.

    Cross-Sectional Custom Sorting & Quantiles: For box plots and histograms, FLAViz includes optional configurations to strip out outliers automatically or filter by custom quantiles (e.g., plotting the wealth distribution of only the top 10% richest households). Our script plots raw agent populations from the data tables without subsetting.

2. Missing Plotting Customizations (Styling & Settings)

The original FLAViz relies on a comprehensive, secondary plot_config.yaml file to control exact Matplotlib properties. Our script currently lacks support for the following plot configuration parameters:

    Logarithmic Axis Scaling (log_scale): Essential for macroeconomic agent distributions (like firm asset sizes or consumer wealth), which typically follow power-law or Pareto distributions.

    Custom Color Palettes & Line Customization: The original allows users to define custom color arrays, hex values, specific line markers, and line thicknesses directly from the config file. Our script uses Matplotlib's default automated color loops.

    Fixed Axes Boundaries (xlim / ylim): The original script allows you to lock plot coordinates across distinct configuration files so that visual spaces are easily comparable side-by-side. Our script relies on Matplotlib’s automatic axis scaling (autoscale).

3. Structural & File Format Capabilities

    Native File Formats: The original FLAViz script was explicitly built to read dense HDF5 binary tables (.h5) or query local SQLite databases using a customized schema created by parsing raw XML model outputs. Our lightweight pipeline completely drops XML/HDF5/SQLite processing to work exclusively with Parquet folders.
